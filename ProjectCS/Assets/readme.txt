○制作ルール（守らないやつは殺す）
3/13（木）作成

[Unityバージョン]
Unity 2022.3.23f1
で制作を予定しているので各自上記バージョンをインストールしておいてください

[フォルダ・ファイル命名規則]
原則、各単語の頭文字を大文字にする方針で統一します
大文字でない・アンダーバーで挟む　などは使いません
（名前がダブったときなどの例外はあるかも）
なお、スクリプトにも言えることですが、英語の綴りミスはやめてください

・命名例
SceneTest　→　○
scenetest,Scenetest,sceneTest,Scene_Test etc.　→　✕
SceenTest,SceneTast　→　✕


[スクリプト作成規則]
原則、スクリプトの初めに必ず
のようにスクリプトの詳細を記載しておいてください
//======================================================
// [スクリプト名]
// 作成者：〇○
// 最終更新日：x/xx
// 
// [Log]
// 3/18　竹内　デバッグログ作成
// 3/19　竹内　.ignoreでやらかす
//======================================================
上記のような感じで記載してもらえたら大体なんでもいいです
つまり、誰がどこをいつ担当したのかを知りたいのでログとして残しておきたいということです
また、各コードには「必ず」コメントを残しておいてください

・コメント例
    // Start出力
    Debug.Log("Start");


[フォルダ構成]
原則、ゲーム本編用のフォルダ（Game）とプログラム作成用のフォルダ（WorkSpace）で
作業場所を分けて作業を行います
（SourceTreeでマージを行う際の競合を避けるため）

流れとしては
１.自身の作業場所で担当する処理や内容を作る
２.完成次第、竹内に報告
３.担当者がWorkProjectからGameへ処理を適用する
４.竹内がSourceTreeでマージを行う
５.ガントチャートから新しい作業を探す
６.１.に戻る
を想定しています

以下は現時点（3/13）でのフォルダ構成です
（使いやすさや意見から変更する場合があります）
Root
	→Game
		→Title：タイトルシーン
		→StageSelect：ステージ選択シーン
		→Stage*：メインシーン（ステージ）
	
	→WorkSpace
		→Takeuchi（自身の名前のフォルダ）
			→Sceneフォルダ
			→Prefabフォルダ
			→Scriptフォルダ
			→etc.（自分の好きなフォルダを用意して整理しておくこと）
		→NameA
		→NameB
		→NameC



[作業時のルール]
なお、作業の際は以下のルールを守ってください
・WorkSpaceに自分の名前のフォルダを作成し、そこに自分の作業を残す
・他の人が見ることを想定して、フォルダやファイルは整理しておく
・スクリプトには必ずコメントを細かく残す
・シーンを跨いで使用したい変数やオブジェクト、その他の処理を共有したい時は竹内に報告して指示を仰ぐこと
・わからないことがあったらすぐに聞くこと
